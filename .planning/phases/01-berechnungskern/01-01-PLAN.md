<objective>
Der framework-unabhängige Berechnungskern von PartyPayback. Aus einer Liste von Personen und Ausgaben berechnet er Gesamtkosten, gleichmäßigen fairen Anteil, Saldo pro Person und konkrete Greedy-Ausgleichszahlungen — und setzt vorher die Fachregeln durch.

Purpose: Der Kern trägt den Core Value des Projekts. Wenn alles andere wegfällt, müssen Salden und Ausgleichszahlungen stimmen. Deshalb ist er reines TypeScript ohne DOM/UI und vollständig per Vitest abgesichert.

Output: Ein in sich geschlossenes `src/core/`-Modul mit Datentypen, Geld-Rundung, Validierung, Settlement-Berechnung und grünen Unit-Tests. Die UI (Phase 2) bindet diesen Kern später über `src/core/index.ts` an.
</objective>

**Typen / Interfaces (in `types.ts`):**
- `Person` — `{ id: string; name: string }`
- `Expense` — `{ id: string; payerId: string; description: string; amount: number }` (amount = Euro, Wert >= 0)
- `PersonBalance` — `{ personId: string; paid: number; fairShare: number; balance: number }` (balance = paid - fairShare; positiv = bekommt zurück, negativ = muss zahlen)
- `Transfer` — `{ fromId: string; toId: string; amount: number }` (fromId zahlt amount an toId)
- `SettlementResult` — `{ totalCost: number; fairShare: number; balances: PersonBalance[]; transfers: Transfer[] }`
- `ValidationError` — `{ code: ValidationErrorCode; message: string }`
- `ValidationErrorCode` — Union: `'TOO_FEW_PERSONS' | 'NEGATIVE_AMOUNT' | 'PAYER_NOT_REGISTERED'`
- `ValidationResult` — `{ ok: true } | { ok: false; errors: ValidationError[] }`

**Funktionen:**
- `money.ts`: `roundEuro(value: number): number` (auf 2 Nachkommastellen, Cent-genau), `toCents(value: number): number`, `fromCents(cents: number): number`
- `validation.ts`: `validateSettlementInput(persons: Person[], expenses: Expense[]): ValidationResult`
- `settlement.ts`: `calculateSettlement(persons: Person[], expenses: Expense[]): SettlementResult` (wirft `SettlementValidationError` bei ungültiger Eingabe), Hilfsfunktionen `calculateTotalCost`, `calculateFairShare`, `calculateBalances`, `calculateTransfers` (exportiert für gezielte Tests)
- `index.ts`: re-exportiert alle öffentlichen Typen und Funktionen

PartyPayback Phase 1 ist eine reine, in-memory TypeScript-Bibliothek: kein Netzwerk, keine Persistenz, kein DOM, keine externe oder Nutzer-I/O-Grenze. Damit existiert in dieser Phase **keine** echte Trust Boundary — die einzige Eingabe sind in-process von Phase 2 übergebene Arrays. Die Angriffsfläche eines Offline-Rechenkerns ist praktisch null.

| Boundary | Description |

| (keine) | Kein Prozess-/Netz-/Speicher-Übergang von untrusted Quellen in dieser Phase. Eingabe ist in-process. |
